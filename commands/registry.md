## Command Development Guidelines

### Standards for New Commands

1. **Clear Purpose**: Single responsibility with specific use case
2. **Bridge Integration**: Compatible with multi-agent coordination framework
3. **Documentation**: Usage examples, parameter descriptions, error handling
4. **Error Handling**: Graceful degradation and helpful error messages
5. **Testing**: Validation in real project context before propagation

### Integration Requirements

- Must work with existing Claude Code `/agents` system
- Should enhance rather than replace existing functionality
- Compatible with bridge system messaging protocols
- Follows framework v2.1+ authority domain principles

### Propagation Criteria

- **High Value**: Significant benefit to multiple projects
- **Proven**: Tested in production context with positive outcomes
- **Generalizable**: Applicable beyond original domain
- **Framework Aligned**: Consistent with collaboration principles
