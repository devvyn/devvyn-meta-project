# Project Structure Guidelines

## Directory Organization

```
project-name/
├── src/           # Source code
├── tests/         # Test files
├── docs/          # Documentation (only when explicitly needed)
├── config/        # Configuration files
└── README.md      # Project overview (only when explicitly requested)
```

## File Naming

- Use kebab-case for directories and files
- Use PascalCase for React components
- Use camelCase for JavaScript/TypeScript functions and variables

## Documentation Policy

- Only create documentation when explicitly requested
- Prefer self-documenting code over extensive comments
- Keep README files minimal and focused
